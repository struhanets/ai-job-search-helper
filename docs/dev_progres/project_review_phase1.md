# AI Job Search Helper — Огляд Фази 1 (Фундамент та Скрейпінг)

У цьому документі зафіксовано поточний стан розробки, архітектурні рішення, створені компоненти та взаємозв'язки між ними.

---

## 🚀 Що вже було реалізовано

Ми заклали міцну архітектурну основу проєкту, налаштували асинхронну взаємодію з API, інтегрували роботу з Gemini 1.5/2.5 Flash та побудували перший набір сервісів для обробки CV й вакансій з DOU.

### 1. Моделі Даних (Domain Models)
Розроблено валідаційні моделі на базі **Pydantic v2**:
*   `CandidateProfile`: Модель розпарсеного резюме (ПІБ, тайтл, роки досвіду, стек, досвід роботи, освіта, мови, резюме).
*   `Vacancy`: Модель вакансії, що містить ідентифікатори, опис, вимоги до кандидатів, вилку зарплат, локації та `raw_text` для подальшої обробки AI.
*   `MatchResult`: Модель оцінки відповідності кандидата вимогам вакансії (score, reasoning, pros/cons).

### 2. Налаштування та Конфігурація (`app/config/settings.py`)
*   Використано `pydantic-settings` для читання та валідації конфігурації з файлу `.env`.
*   Налаштовано параметри для роботи з AI (GEMINI_API_KEY, модель, температура).
*   Налаштовано параметри скрейпінгу (User-Agent, затримки для запобігання блокувань).

### 3. Універсальний AI Клієнт (`app/ai/client.py`)
*   Створено асинхронний `AIClient` на базі актуального SDK `google-genai` (використовуючи `from google import genai`).
*   Реалізовано метод `get_structured_data()`, який приймає Pydantic-модель і за допомогою параметра `response_schema` змушує Gemini повертати **гарантовано валідований JSON-об'єкт**.

### 4. Парсер резюме (`app/parsers/` та `app/services/`)
*   **`PdfParser`** (`pdf_parser.py`): Використовує високошвидкісну бібліотеку `PyMuPDF` (`fitz`) для точного вилучення тексту з PDF (зберігає структуру колонок).
*   **`ResumeParserService`** (`cv_parser.py`): Отримує текст з PDF, додає детальний системний промпт (правила оцінки комерційного досвіду, виокремлення навчання від реальної роботи) та надсилає запит до `AIClient` для повернення готового `CandidateProfile`.

### 5. Модуль скрейпінгу DOU (`app/scrapers/dou/`)
Ми розділили логіку скрейпінгу на 3 компоненти відповідно до кращих практик:
*   **`DouClient`** (`dou_client.py`): Відповідає суто за мережеву взаємодію (асинхронно через `httpx.AsyncClient`).
    *   `fetch_jobs_page(query)`: Завантажує сторінку пошуку DOU за технологією (напр. category=Python).
    *   `fetch_single_page(url)`: Завантажує детальну сторінку конкретної вакансії.
*   **`DouParser`** (`dou_parser.py`): Використовує `BeautifulSoup4` для обробки HTML.
    *   `parse_vacancy(html)`: Витягує первинний список вакансій (ID, назву, компанію, локацію та посилання).
    *   `parse_single_vacancy(html, base_vacancy)`: Витягує повний внутрішній опис вакансії (`raw_text`) для збагачення моделі.
*   **`DouScraper`** (`dou_scraper.py`): Оркеструє роботу клієнта і парсера. Робить запит на список вакансій, бере перші кілька вакансій, довантажує для кожної детальний опис та повертає збагачений список об'єктів `Vacancy`.

### 6. API Роути (`app/api/routers/`)
*   `/health`: Перевірка працездатності додатку.
*   `/test` (в `ai.py`): Тестовий роут для перевірки інтеграції з Gemini.
*   `/resumes/parse` (в `cv.py`): Приймає PDF-файл резюме та повертає його структурований AI-аналіз.
*   `/jobs/search` (в `jobs.py`): Ендпоінт для запуску скрейпінгу вакансій.

## 🛠️ Наступні кроки (План на Фазу 2)

Тепер, коли ми маємо дані про кандидата та детальні описи вакансій з вебу, ми готові будувати **інтелектуального агента**:

1.  **Створення `JobMatchingService`:** 
    *   Написати промпт для AI, який порівнюватиме `CandidateProfile` та `Vacancy` (з її `raw_text`).
    *   AI аналізуватиме hard skills, soft skills, досвід, мови та вираховуватиме `MatchResult` (score від 0 до 100, плюси, мінуси, рекомендації).
2.  **Генерація Excel-звіту:**
    *   Написати утиліту (використовуючи `pandas` або `openpyxl`), яка зберігатиме результати збігів у красивий `.xlsx` файл (назва компанії, роль, лінк, оцінка відповідності, резюме від AI).
3.  **Автоматична відправка листів (Етап 2 проєкту):**
    *   Якщо оцінка відповідності висока (наприклад, > 85%), агент формує персоналізований Cover Letter під цю вакансію.
